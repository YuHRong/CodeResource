HTML 基础知识回顾

HTML基础知识

  • HTML 的作用：HTML 代表网页的内容和结构。
  • HTML元素：元素是HTML文档的构建模块。它们代表标题、段落、链接、图像等等。大多数HTML元素由一个开始标签（<a> <elementName>）和一个结束标签（<br> </elementName>）组成。

以下是基本语法：

<elementName>Content goes here</elementName>
  • 空元素：空元素不能包含任何内容，只能包含一个开始标签。例如 `<div>`img和 `<span> meta` 元素。
<img>
<meta>

/有些代码库会在 void 元素内包含正斜杠。以下两种情况都是可以接受的：

<img>
<img/>
  • 属性：属性是放置在 HTML 元素开始标签内的值。属性提供有关元素的附加信息，或指定元素的行为方式。以下是属性的基本语法：
<element attribute="value"></element>

布尔属性是指HTML标签中可以存在或不存在的属性。如果存在，则值为true，否则为false。布尔属性的示例包括`<a>` disabled、readonly`<b>`和`<c> required`。

  • 注释：注释用于在编程中给自己和其他开发者留下代码注释。以下是 HTML 注释的语法：
<!--This is an HTML comment.-->

标识符和分组

  • ID：HTML元素的唯一标识符。每个HTML文档中，ID名称只能使用一次。
<h1 id="title">Movie Review Page</h1>

ID名称不能包含空格。如果您的ID名称包含多个单词，则可以使用短横线连接单词，例如：

<div id="red-box"></div>
  • 类：类用于对元素进行分组，以便设置样式和行为。
<div class="box"></div>

与 ID 不同，您可以在整个 HTML 文档中重复使用相同的类名。该class值也可以包含空格，例如：

<div class="box red-box"></div>
<div class="box blue-box"></div>

特殊字符和链接

  • HTML 实体：HTML 实体，或字符引用，是一组用于表示 HTML 中保留字符的字符。例如，&&amp;符号和小于号 (" &lt;")。
<p>This is an &lt;img /&gt; element</p>
  • link元素：此元素用于链接到外部资源，例如样式表和网站图标。以下是使用此link元素链接到外部 CSS 文件的基本语法：
<link rel="stylesheet" href="./styles.css" />

该rel属性用于指定链接资源与 HTML 文档之间的关系。该href属性用于指定外部资源的 URL 位置。

  • script元素：此元素用于嵌入可执行代码。
<body>
  <script>
    alert("Welcome to freeCodeCamp");
  </script>
</body>

虽然理论上你可以将所有 JavaScript 代码都写在标签内，但最佳实践是链接到外部 JavaScript 文件。以下是使用`<script>` 元素链接到外部 JavaScript 文件script的示例：script

<script src="path-to-javascript-file.js"></script>

该src属性用于指定外部 JavaScript 文件的位置。

  • head元素：该head部分包含重要的元数据，这些元数据是浏览器和搜索引擎所需的后台信息。
  • meta元素：这些元素代表您网站的元数据。这些元素包含字符编码、Twitter 等网站如何预览您页面的链接等详细信息。
  • title元素：此元素用于设置显示在浏览器标签页或窗口中的文本。
  • UTF-8 字符编码：UTF-8，即 UCS 转换格式 8，是一种广泛应用于 Web 的标准化字符编码。字符编码是计算机用于存储字符数据的方法。`<code>`元素charset内部的 `encoding` 属性用于meta将字符编码设置为 UTF-8。

目标属性类型

  • target属性：此属性告诉浏览器在哪里打开锚元素的 URL。此属性有四个重要的可能值：`false`、`false` _self、 `false`和`false`。还有第五个值，称为`false`，目前用于实验性API。您可能暂时不需要使用它。_blank_parent_top_unfencedTopFencedFrame
  • _self值：这是该属性的默认值target。此值会在当前浏览上下文中打开链接。在大多数情况下，这将是当前标签页或窗口。

  • _blank值：此操作会在新的浏览器环境中打开链接。通常情况下，链接会在新标签页中打开。但有些用户可能将浏览器配置为在新窗口中打开。
<a href="https://freecodecamp.org" target="_blank">Visit freeCodeCamp</a>
  • _parent值：此操作会在当前上下文的父级元素中打开链接。例如，如果您的网站包含 iframe，则_parent该 iframe 中的值会在您网站的标签页/窗口中打开，而不是在嵌入的 iframe 中打开。
<a href="https://freecodecamp.org" target="_parent">Visit freeCodeCamp</a>
  • _top值：这会在最顶层的浏览器上下文中打开链接——可以理解为“父级的父级”。这与类似_parent，但链接始终会在完整的浏览器标签页/窗口中打开，即使对于嵌套的嵌入式框架也是如此。
<a href="https://freecodecamp.org" target="_top">Visit freeCodeCamp</a>

绝对路径与相对路径

  • 路径定义：路径是一个字符串，用于指定文件或目录在文件系统中的位置。在Web开发中，路径允许开发人员链接到图像、样式表、脚本和其他网页等资源。
  • 路径语法：需要了解三种关键的语法。首先是斜杠，根据操作系统不同，可以是反斜杠 ( \) 或正斜杠 ( )。其次是单点 ( )。最后是双点 ( )。斜杠被称为“路径分隔符”，用于指示文件夹或文件名之间的分隔符。单点指向当前目录，双点指向父目录。/...
public/index.html
./favicon.ico
../src/index.css
  • 绝对路径：绝对路径是指向资源的完整链接。它从根目录开始，包含所有其他目录，最后是文件名和扩展名。“根目录”指的是层级结构中的顶级目录或文件夹。绝对路径还包含协议（例如 `http://`、`http://`http和`http://`）https以及file域名（如果资源位于网络上）。以下是一个指向 freeCodeCamp 徽标的绝对路径示例：
<a href="https://design-style-guide.freecodecamp.org/img/fcc_secondary_small.svg">

  • 相对路径：相对路径指定文件相对于当前文件所在目录的位置。它不包含协议或域名，因此更短，也更灵活，适用于同一网站内的内部链接。以下示例展示了如何从about.html页面链接到contact.html位于同一文件夹中的页面：

链接状态

  • :link这是默认状态。此状态表示用户尚未访问、点击或与之交互的链接。您可以将此状态理解为页面上所有链接的基础样式。其他状态均基于此状态构建。
  • :visited：这适用于用户已访问过链接指向的页面的情况。默认情况下，链接会变成紫色，但您可以使用 CSS 为用户提供不同的视觉指示。
  • :hover当用户将鼠标悬停在链接上时，此状态生效。此状态有助于提高用户对链接的关注度，确保用户确实打算点击该链接。
  • :focus：当我们聚焦于某个链接时，就会出现这种状态。
  • :active此状态适用于用户激活的链接。通常情况下，这意味着用户使用鼠标左键点击链接。
