第五节： 练习 - 发现引用类型 

引用类型包括数组、类和字符串。 关于应用程序执行时存储值的方式，引用类型与值类型的处理方式不同。
在此练习中了解引用类型与值类型的不同之处，以及如何使用

引用类型与值类型有何不同  
值类型变量将其值直接存储在名为堆栈的存储区域中。
 堆栈是为 CPU 上当前运行的代码分配的内存（也称为堆栈帧或激活帧）。 堆栈帧执行完毕后，堆栈中的值将被删除。
引用类型变量将其值存储在名为堆的单独内存区域中。
 堆是一个内存区域，由操作系统上运行的多个应用程序同时共享。 .NET 运行时与操作系统进行通信以确定可用的内存地址，并请求可存储该值的地址。 .NET 运行时会存储值，然后将内存地址返回给变量。 当代码使用变量时，.NET 运行时会无缝查找变量中存储的地址，并检索其中存储的值。
接下来你将编写一些代码来更清楚地说明这些概念。

选择适当的数据类型  
由于可供选择的数据类型太多，针对特定情况应采用哪种标准选择适当的数据类型呢？
在评估这些选项时，必须考虑一些重要的注意事项。 正确答案通常不止一个，但有些答案比其他答案更正确。

选择满足应用程序的边界值范围要求的数据类型  
所选的数据类型可以帮助设置该特定变量中可能存储的数据大小的边界。 例如，如果知道某个特定变量只能存储 1 - 10,000 之间的值，否则将超出预期边界，则可能要避免使用

long
uint

double
和

首先选择适合数据的数据类型（不优化性能）  
你可能会倾向于选择使用最少位来存储数据的数据类型，认为这样可提高应用程序的性能。 但某些与应用程序性能（即，应用程序的运行速度）相关的最佳建议却是不要“过早优化”。 应该阻止自己猜测代码的哪部分（包括数据类型的选择）可能会影响应用程序的性能。
很多人认为，因为给定的数据类型存储的信息较少，所以使用的计算机处理器和内存一定会少于存储更多信息的数据类型。 其实应该选择适合数据的数据类型，然后可以凭经验使用特殊软件来衡量应用程序的性能，以真正地了解应用程序的哪部分可能会对性能产生负面影响。

根据所用库函数的输入和输出数据类型选择数据类型  标题3
假设你想要处理在两个日期之间间隔的年份范围。 由于该应用程序是一个业务应用程序，你可能会确定只需 1960 到 2200 的范围。 你可能会考虑尝试使用
byte
，因为它可以表示 0 - 255 之间的数字。
但当你研究
System.TimeSpan
和
System.DateTime

double

int

sbyte
，则会在
byte

double

int

int
会更有意义，如果需要亚秒级精度，则选择
double

根据对其他系统的影响选择数据类型  
有时必须考虑其他应用程序或其他系统（如数据库）使用信息的方式。 例如，SQL Server 的类型系统不同于 C# 的类型系统。 因此，必须在这两个系统之间进行某些映射之后才能将数据保存到该数据库。
如果应用程序的目的是与数据库进行交互，则你可能需要考虑数据的存储方式和存储的数据量。 选择较大的数据类型可能会影响存储应用程序将生成的所有数据所需的物理存储的量（和成本）。

如有疑问，请继续使用基本类型  标题3
虽然你已经考虑了一些注意事项，但在开始使用时，为简单起见，应首选部分基本数据类型，包括：

decimal
，适用于表示资金的数字
bool
，适用于 true 或 false 值
string
，适用于字母数字值

针对特殊情况选择专门的复杂类型 
如果已存在一个或多个数据类型用于给定目的，请勿重新创建数据类型。 以下示例确定了特定 .NET 数据类型的有用之处：
byte
：用于来自其他计算机系统或使用不同字符集的编码数据。
double
：用于几何学或科研用途。
double
常用于生成涉及运动的游戏。
System.DateTime

System.TimeSpan
，适用于年/月/日/小时/分钟/秒/毫秒范围。

为代码选择数据类型时，需要考虑一些事项，通常不止一个选项。 仔细考虑你的选择，除非你有充分的理由，否则尽量继续使用基本类型，如
