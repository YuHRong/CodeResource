C#中的变异允许你使用更灵活的委托方法签名。
它使你能够将方法分配给委托，即使它们的参数类型或返回类型不完全相同，只要它们遵循某些规则。
变异在处理继承层次结构时很有用。

## 什么是协变？

协变是 C#中的一个概念，它允许你使用比委托或接口中指定的更派生类型。<br>它通过允许方法分配给委托即使它们的签名不完全匹配，从而创建更灵活和可重用的代码。

协变有两种类型：
- `协变`：允许方法的返回类型比委托中定义的类型更派生。
- `逆变`：允许方法接受比委托类型参数更不派生的参数。

协变支持包括泛型和非泛型委托。

## 自定义委托的协变

自定义委托是用户定义的委托类型。
可以为自定义委托应用变体，使其更加灵活。

使用自定义委托的变体示例：

```c#
public class Animal { }
public class Dog : Animal { }

// Define a delegate that takes a Dog and returns an Animal
public delegate Animal AnimalDelegate(Dog dog);

// Method that matches the delegate signature
public static Animal GetAnimal(Dog dog) => new Animal();

// Method that uses covariance (returns a more derived type)
public static Dog GetDog(Dog dog) => new Dog();

// Method that uses contravariance (accepts a less derived type)
public static Animal GetAnimalFromAnimal(Animal animal) => new Animal();

public class Program
{
    public static void Main()
    {
        AnimalDelegate del;

        // Assign method with matching signature
        del = GetAnimal;
        Animal animal = del(new Dog());

        // Assign method with covariant return type
        del = GetDog;
        animal = del(new Dog());

        // Assign method with contravariant parameter type
        del = GetAnimalFromAnimal;
        animal = del(new Dog());
    }
}
```

在这个示例中，`GetDog`可以赋值给`AnimalDelegate`，因为`Dog`是比`Animal`更派生的类型（协变）。类似地，`GetAnimalFromAnimal`也可以赋值，因为`Animal`是比`Dog`更不派生的类型（逆变）。

## 强类型委托的协变

协变也适用于强类型委托，如`Action`和`Func`。

### 与`Func`的协变

与`Func`的协变示例：

```c#
public class Person { }
public class Employee : Person { }

public static Employee FindEmployee(string title) => new Employee();

public class Program
{
    public static void Main()
    {
        // Func<string, Person> can hold a method that returns Employee
        Func<string, Person> func = FindEmployee;
        Person person = func("Manager");
    }
}
```

在这个示例中，`FindEmployee`返回一个`Employee`，它比`Person`更具体，所以它可以赋值给`Func<string, Person>`。

### 与`Action`的逆变

与`Action`的逆变示例：

```c#
public class Animal { }
public class Dog : Animal { }

public static void HandleAnimal(Animal animal) { }

public class Program
{
    public static void Main()
    {
        // Action<Dog> can hold a method that takes Animal
        Action<Dog> action = HandleAnimal;
        action(new Dog());
    }
}
```

在这里，`HandleAnimal`接受一个`Animal`，它比`Dog`派生程度低，因此可以赋值给`Action<Dog>`。

## 泛型中的变体

变体也可以使用`in`和`out`关键字应用于泛型类型参数。

### 协变泛型类型

`out`关键字用于声明协变泛型类型参数。这项功能允许你将更派生化的类型用作返回类型。
演示泛型中的协变示例：

```c#
public interface ICovariant<out T> { T Get(); }

public class Animal { }
public class Dog : Animal { }

public class CovariantExample : ICovariant<Dog>
{
    public Dog Get() => new Dog();
}

public class Program
{
    public static void Main()
    {
        ICovariant<Animal> covariant = new CovariantExample();
        Animal animal = covariant.Get();
    }
}
```

在这个示例中，`ICovariant<out T>`允许`ICovariant<Dog>`赋值给`ICovariant<Animal>`，因为`Dog`比`Animal`更具派生性。

### 逆变泛型类型

`in`关键字用于声明逆变泛型类型参数。此功能允许你将一个派生程度较低的类型用作参数类型。

演示泛型中的逆变示例：

```c#
public interface IContravariant<in T> { void Set(T value); }

public class Animal { }
public class Dog : Animal { }

public class ContravariantExample : IContravariant<Animal>
{
    public void Set(Animal value) { }
}

public class Program
{
    public static void Main()
    {
        IContravariant<Dog> contravariant = new ContravariantExample();
        contravariant.Set(new Dog());
    }
}
```

这里，`IContravariant<in T>`允许`IContravariant<Animal>`赋值给`IContravariant<Dog>`，因为`Animal`的派生程度低于`Dog`。

## 总结

- 协变允许你使用更灵活的委托方法签名。
- 协变允许方法具有比委托中定义的类型更派生的返回类型。
- 逆变允许方法接受比委托类型参数更不派生的参数。
- 自定义和强类型委托可以使用协变来提高灵活性。
- 泛型可以使用`in`和`out`关键字来声明逆变和协变的泛型类型参数。
