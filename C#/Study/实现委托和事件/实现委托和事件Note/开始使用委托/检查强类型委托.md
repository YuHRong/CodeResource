在 C#编程中，委托提供了一种封装方法并作为参数传递的方式，从而实现灵活且可重用的代码。  
然而，传统委托在处理时可能比较繁琐，因为它们需要为每种委托类型都进行显式定义。  
如果你需要为不同的方法签名创建多个委托类型，这会导致大量样板代码，并使代码库更难维护。  

幸运的是，  
.NET 框架提供了一种比传统委托更灵活的替代方案：强类型委托。

强类型委托通过允许您使用泛型而不是为每个方法签名定义自定义委托类型，简化了使用委托的过程。

强类型委托使您能够创建一个可以与多种方法签名协同工作的委托类型。

以下是一些 C#开发者常用的强类型委托：
- `Action`: 当您需要执行不返回值的操作时，请使用`Action`。
- `Func`: 当您需要执行返回值的操作时，请使用`Func`。
- `Predicate`: 使用`Predicate`为执行测试并返回 `true` 或 `false` 的方法。
- `Comparison`: 使用`Comparison`为比较两个对象并返回一个整数指示它们相对顺序的方法。
- `Converter`: 使用`Converter`为将对象从一种类型转换为另一种类型的方法。

## 检查`Action`和`Func`代表

`Action`和`Func`是泛型委托，可用于定义不同签名的无需自定义委托类型的方法。  
这些委托提供了以下好处：
- `易用性`：使用`Action`和`Func`，您可以快速创建委托，而无需担心定义自定义类型。<br>这使得您的代码更加简洁易懂。
- `一致性`：使用预定义的委托类型可确保整个代码库的一致性，因为开发人员使用相同的委托类型来完成相似的任务。
- `提升可读性`：使用`Action`和`Func`的代码通常更易读，因为委托类型清楚地表明了参数的数量和类型，以及返回类型。
- `增强可维护性`：当代码更易于阅读且一致时，它就更容易被维护和调试。<br>在包含多个开发者的大型项目中，增强可维护性非常重要。

## 检查`Action`委托

`Action`委托代表一个最多接受 16 个参数且不返回值的方法。<br>当您想执行操作但不需要返回结果时使用。

以下代码演示了如何定义无参数、一个参数和两个参数的`Action`委托：

```c#
Action action;
Action<int> actionWithOneParameter;
Action<int, string> actionWithTwoParameters;
```

在这个代码中，`action`是一个不接受参数的委托，`actionWithOneParameter`是一个接受一个整数参数的委托，而`actionWithTwoParameters`是一个接受一个整数和一个字符串参数的委托。

一个`Action`代理可以用来替换一个用于执行相似操作的定制代理类型。<br>这种能力在你想要将方法作为参数传递或分配给变量，而不需要定义自定义委托时很有用。`Action`委托可以减少样板代码的量，使代码更加简洁易读。

考虑以下代码，它定义并实现了一个名为`PrintMessageDelegate`的自定义委托类型来打印消息：

```c#
public delegate void PrintMessageDelegate(string message);

public class TestClass
{

    static void Main()
    {
        PrintMessageDelegate printMessage = PrintMessage;
        printMessage("Hello, World!"); // Output: Hello, World!
    }

    static void PrintMessage(string message)
    {
        Console.WriteLine(message);
    }
}
```

这段代码定义了一个 `PrintMessage` 方法，该方法接受一个字符串参数（ `message` ），并将其打印到控制台。 

`PrintMessageDelegate`委托类型被定义以匹配`PrintMessage`方法的签名。在`Main`方法中，创建了一个`PrintMessageDelegate`的实例并将其分配给`PrintMessage`方法。当`printMessage("Hello, World!")`被调用时，它将"`Hello, World!`"打印到控制台。

与其定义一个自定义委托类型，您可以使用`Action`委托以更少的代码实现相同的结果：

```c#
public class TestClass
{
    static void Main()
    {
        Action<string> printMessage = delegate(string message)
        {
            Console.WriteLine(message);
        };
        printMessage("Hello, World!"); // Output: Hello, World!
    }
}
```

这段代码定义了一个名为`printMessage`的`Action`委托，它接受一个字符串参数。  
委托被分配了一个匿名方法，该方法将消息打印到控制台。

当`printMessage("Hello, World!")`被调用时，它将"Hello, World!"打印到控制台。

一个`Action`代表也可以用来对参数执行操作。<br>例如，您可以使用一个`Action`委托来反转字符串并将其打印到控制台：

```c#
        for (int i = charArray.Length - 1; i >= 0; i--)
        {
        Console.Write(charArray[i]);
        }
        Console.WriteLine();
    };
    reversePrint("CSharp"); // Output: prahSC
}
```

该委托被分配了一个匿名方法，该方法反转字符串并将其打印到控制台。

当`reversePrint("CSharp")`被调用时，它将打印 "`prahSC`" 到控制台。

前两个示例可以使用 `lambda 表达式`来简化，`lambda 表达式`为定义委托提供了更简洁的语法。<br>`Lambda 表达式`是 C#的一项强大功能，它允许你以更易读的方式定义匿名方法。

一个无参数的`Action`委托也可以使用 `lambda 表达式`来定义。

以下代码演示了一个无参数的`Action`委托：

```c#
Action sayHello = () => Console.WriteLine("Hello, World!");
sayHello(); // Output: Hello, World!
```

这个示例声明了一个不带参数的`Action`。lambda 表达式`() => Console.WriteLine("Hello, World!")`定义了`Action`指向的方法。当`sayHello()`被调用时，它向控制台打印"`Hello, World!`"。

以下代码演示了一个带有两个整数参数的`Action`委托：

```c#
Action<int, int> printSum = (a, b) => Console.WriteLine($"Sum: {a + b}");
printSum(5, 7); // Output: Sum: 12
```

这段代码定义了一个名为`printSum`的`Action`委托，它接受两个整数参数（`a`和`b`）。

委托被分配了一个计算两个整数之和并打印结果的 `lambda 表达式`。

当`printSum(5, 7)`被调用时，它向控制台打印"`Sum: 12`"。

### 检查`Func`委托

一个`Func`代表一个最多接受 16 个参数并返回值的方法。

当您需要执行操作并获取结果时，会使用`Func`委托。

定义`Func`委托的语法与`Action`委托相似，但最后一个类型参数指定了返回类型。

前 16 个类型参数指定输入参数。  
以下代码演示了如何定义无参数、一个参数和两个参数的`Func`委托：

```c#
Func<int> funcWithNoParameters;
Func<int, string> funcWithOneParameter;
Func<int, string, bool> funcWithTwoParameters;
```

这段代码定义了一个无参数的`Func`委托（`funcWithNoParameters`返回一个`int`），一个参数的委托（`funcWithOneParameter`返回一个`string`），以及一个有两个参数的委托（`funcWithTwoParameters`返回一个`bool`）。最后一个类型参数指定了委托的返回类型。

考虑以下代码，它定义并实现了一个名为`CompareCustomers`的自定义委托类型。`CompareCustomers`委托接受两个`Customer`对象作为参数，并返回一个整数来表示它们的相对顺序。

比较所使用的标准在委托实例化时使用 `lambda 表达式`定义。

```c#
public delegate int CompareCustomers(Customer c1, Customer c2);
public class TestClass
{
    static void Main()
    {
        Random random = new Random();
        List<Customer> customers = new List<Customer>();

        // Create 10 customers with random ages and IDs
        for (int i = 0; i < 10; i++)
        {
            string customerId = random.Next(10000000, 99999999).ToString();
            int age = random.Next(30, 51); // Random age between 30 and 50
            customers.Add(new Customer(customerId, age));
        }

        // Define the CompareCustomers delegate
        CompareCustomers compare = (c1, c2) => c2.Age.CompareTo(c1.Age);
        customers.Sort((c1, c2) => compare(c1, c2));

        // Print sorted customers
        foreach (var customer in customers)
        {
            Console.WriteLine($"ID: {customer.CustomerId}, Age: {customer.Age}");
        }
    }
}
public class Customer
{
    public string CustomerId { get; set; }
    public int Age { get; set; }

    public Customer(string custId, int age)
    {
        CustomerId = custId;
        Age = age;
    }
}
```

这段代码定义了一个`Customer`类，包含`CustomerId`和`Age`属性。
`Main`方法创建了一个包含 10 名客户的列表，客户的年龄和 ID 是随机的。在文件顶部定义了一个`CompareCustomers`委托，然后使用 `lambda 表达式`实例化它，以定义排序标准。

lambda 表达式`(c1, c2) => c2.Age.CompareTo(c1.Age)`比较两个`Customer`对象的`Age`属性。`CompareTo`方法如果
`c2`比`c1`年长，则返回正数；如果`c1`比`c2`年长，则返回负数；如果它们年龄相同，则返回零。

这意味着客户将按年龄降序排序。

`customers.Sort`方法使用一个 `lambda 表达式`，该表达式使用`compare`委托来按年龄降序对客户进行排序。

使用委托可以让你轻松地从按`Age`排序切换到按`CustomerId`排序。例如，以下 `lambda 表达式`可用于按`CustomerId`排序。

```c#
CompareCustomers compare = (c1, c2) => string.Compare(c1.CustomerId, c2.CustomerId, StringComparison.Ordinal);
```

与其定义一个自定义委托类型，您可以使用`Func`委托以更少的代码实现相同的结果：

```c#
public class TestClass
{
    static void Main()
    {
        Random random = new Random();
        List<Customer> customers = new List<Customer>();

        // Create 10 customers with random ages and IDs
        for (int i = 0; i < 10; i++)
        {
            string customerId = random.Next(10000000, 99999999).ToString();
            int age = random.Next(30, 51); // Random age between 30 and 50
            customers.Add(new Customer(customerId, age));
        }

        // Define the Func delegate for comparison
        Func<Customer, Customer, int> compare = (c1, c2) => c2.Age.CompareTo(c1.Age);
        customers.Sort((c1, c2) => compare(c1, c2));

        // Print sorted customers
        foreach (var customer in customers)
        {
            Console.WriteLine($"ID: {customer.CustomerId}, Age: {customer.Age}");
        }
    }
}

public class Customer
{
    public string CustomerId { get; set; }
    public int Age { get; set; }

    public Customer(string custId, int age)
    {
        CustomerId = custId;
        Age = age;
    }
}
```

这段代码定义了一个名为`compare`的`Func`委托，它接受两个`Customer`
对象作为参数，并返回一个整数来指示它们的相对顺序。

委托被分配了一个比较两个`Customer`对象的`Age`属性的 `lambda 表达式`。
`customers.Sort`方法被调用，使用`compare`委托根据年龄降序对客户进行排序。当`customers.Sort`被调用时，它根据年龄降序对客户进行排序。  
排序后的客户将被打印到控制台。

要按`CustomerId`对客户进行排序，您只需更改分配给`compare`委托的 `lambda 表达式`：

```c#
Func<Customer, Customer, int> compare = (c1, c2) => string.Compare(c1.CustomerId, c2.CustomerId, StringComparison.Ordinal);
```

## 总结

强类型委托，如`Action`和`Func`，通过允许您使用泛型而不是为每个方法签名定义自定义委托类型，简化了委托的使用过程。
- `Action`委托用于不返回值的方法，而`Func`委托用于返回值的方法。
- 使用`Action`和`Func`委托减少了样板代码，提高了可读性，并增强了可维护性。
- Lambda 表达式为定义委托提供了简洁的语法，使代码更易读且更容易理解。
- 强类型委托通常可以用来替代自定义委托类型，使代码更加简洁易读。
