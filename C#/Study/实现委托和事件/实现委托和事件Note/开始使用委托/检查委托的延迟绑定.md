 在 C#中，方法用于执行操作，例如执行计算或检索信息。
您的应用程序使用方法来实现预期结果，并在特定条件满足时调用这些方法。
应用程序通常通过一个方法调用另一个方法来自动化流程，后者可能调用第三个方法，依此类推。  
由于条件在编译时已知，可执行代码以可预测的方式实现预定义的工作流程。  
例如，你可能会有一个方法，当点击按钮时生成月度客户报告。
当代码运行且按钮被点击时，该方法被调用，报告将被生成。  
这个场景是早期绑定的一个例子，其中要调用的方法在编译时确定。
早期绑定适用于许多场景，因为它使代码易于阅读和理解。
通过跟踪方法调用，你可以看到程序的流程。  
代码是可预测的，你知道当按钮被点击时具体调用了哪个方法。  
然而，有时你直到程序运行时才知道要调用哪种方法。  
例如，假设你正在开发一个依赖运行时依赖项来决定分析客户数据需要哪种方法的程序。
这种情况是延迟绑定的一个例子，其中要调用的方法在运行时确定。  
在这种情况下，要调用的方法可能取决于外部资源、用户输入或应用程序的当前状态。

在 C#中，委托用于实现延迟绑定。  

## 什么是委托？  

委托是一种派生于`Delegate`类的.NET 类型，用于封装方法。  
通过封装方法并将其实例化为对象，委托使您能够将方法存储在变量中，作为参数传递给其他方法，并在稍后调用它们。  
委托的签名定义了可以分配给它的方法的参数和返回类型。
这意味着你可以创建一个与某个方法的签名匹配的委托类型，然后将任何具有该签名的方 法分配给该委托。
这种能力允许你在编译时不知道具体是哪个方法的情况下，通过委托来调用该方法。  
以下示例展示了如何定义一个委托类型：

```c#
public delegate int PerformCalculation(int x, int y);
```

这个示例定义了一个名为`PerformCalculation`的委托。`PerformCalculation`委托被定义为表示接受两个`int`参数（x 和 y）并返回`int`的方法。

这个委托可以用来封装执行不同计算的方法，例如加法、减法、乘法、除法，或使用两个参数的更复杂的方程式。

例如，这个委托可以用来封装以下方法：

```c#
public class Calculator
{
    public int Add(int x, int y)
    {
        return x + y;
    }

    public int Subtract(int x, int y)
    {
        return x - y;
    }

    public int Multiply(int x, int y)
    {
        return x * y;
    }

    public int Divide(int x, int y)
    {
        if (y == 0)
            throw new DivideByZeroException();
        return x / y;
    }
}
```

这个示例定义了一个`Calculator`类，其中包含的方法与`PerformCalculation`委托的签名相匹配。

你可以创建委托的实例，并将它们分配给不同的方法，这样你就可以根据需要，在运行时调用适当的方法。

以下代码展示了如何使用`PerformCalculation`委托：

```C#
public class Program
{
    public static void Main()
    {
        Calculator calculator = new Calculator();

        // Create delegate instances
        PerformCalculation add = new PerformCalculation(calculator.Add);
        PerformCalculation subtract = new PerformCalculation(calculator.Subtract);
        PerformCalculation multiply = new PerformCalculation(calculator.Multiply);
        PerformCalculation divide = new PerformCalculation(calculator.Divide);

        // Call the methods using the delegates
        Console.WriteLine("Addition: " + add(5, 3)); // Output: 8
        Console.WriteLine("Subtraction: " + subtract(5, 3)); // Output: 2
        Console.WriteLine("Multiplication: " + multiply(5, 3)); // Output: 15
        Console.WriteLine("Division: " + divide(5, 3)); // Output: 1
    }
}
```

这个示例创建了一个`Calculator`类的实例，并将方法分配给`PerformCalculation`委托的实例。
然后使用委托实例调用这些方法，允许你根据分配的委托执行不同的计算。
## 委托类型特征

委托是引用类型，这意味着它们存储在堆上，并且可以像任何其他对象一样传递。  
当你创建一个委托实例时，你正在创建一个封装了方法的对象。  
这种封装使你能够将委托视为一等对象，就像 C#中的任何其他对象一样。  
你可以创建委托的实例，将它们赋值给变量，并将它们作为参数传递给方法。  
这个过程与你在 C#中处理其他对象（如字符串或列表）的方式类似。  

委托类型是密封的，它们不能被派生，并且不可能从`Delegate`类派生自定义类。

委托也是不可变的，这意味着一旦创建委托，你就不能改变它封装的方法。  
但是，你可以创建指向不同方法的新委托实例。

这种不可变性确保委托始终指向相同的方法，这对于维护代码的完整性非常重要。  
你像调用方法一样调用委托实例，它会调用它封装的方法。  
代表具有以下特征：
- 委托使你能够将方法作为参数传递给其他方法。
- 委托可以链接在一起，例如在单个事件上调用多个方法。
- 委托是类型安全的，这意味着编译器会在编译时检查方法签名是否与委托签名匹配。

*注*方法不必与委托类型完全匹配。

如果方法签名兼容，编译器允许将方法分配给委托。这种行为被称为协变和逆变。  
协变允许你使用比原来指定的更派生的类型，而逆变允许你使用更少派生的类型。

这项功能在处理继承和多态时很有用，因为它允许你创建更灵活和可重用的代码。

## 为什么要使用委托？

委托相比直接方法调用有多个优势，并解决了与代码灵活性、动态方法调用和类型安全相关的一些重要问题。  
委托提供了许多好处，包括以下项目：
- `灵活性`：委托允许您将不同的方法作为参数传递，从而根据运行时条件实现动态行为。  
这种灵活性在运行时确定要执行的确切操作时很有用。
- `扩展性`：通过委托，您可以轻松地通过添加新操作来扩展功能，而无需修改现有代码。  
你可以传递任何符合委托签名的任何方法。
- `解耦`：委托将方法调用与方法定义解耦，使代码更具模块化且更易于维护。

委托是 C#的一项强大功能，可以解决编程中的若干常见问题。  
它们提供了一种灵活且类型安全的方法来封装方法，支持动态方法调用、回调方法和多播调用。  
通过使用委托，您可以编写更可重用和可维护的代码，使其能够适应不断变化的需求。

### 动态方法调用

动态方法调用允许你编写更灵活和可重用的代码，因为你可以在编译时不知道它们的具体实现的情况下，将不同的方法作为参数传递。

`场景`：假设你有一个客户列表，并且你需要根据不同的标准对它们进行排序，例如姓名、账户类型或客户 ID。

如果没有委托，你需要为每个标准编写单独的排序方法，这将导致代码重复且难以维护。

`解决方案(Solution)`：委托允许您将方法作为参数传递，从而实现动态方法调用。

你可以定义一个与排序方法签名匹配的委托，并在运行时传递不同的比较函数。  
这使得你的代码更加灵活和可重用。

### 回调方法

回调方法的需求出现在你希望在某个操作完成后执行特定动作的场景中。
回调模式在异步编程中很常见，你可能在长时间运行的任务完成后通知用户或执行其他动作。

`场景`：在异步编程中，当操作完成后，你通常需要执行其他操作。  
如果没有委托，您将不得不将异步操作与后续操作紧密耦合，从而降低灵活性并使代码更难维护。  
`解决方案(Solution)`：委托可以实现回调方法的实现，允许您动态指定后续操作。  
这使异步操作与回调逻辑解耦，增强了灵活性和可维护性。

### 类型安全

类型安全是编程中的一个关键方面，它确保你调用的方法与预期的签名相匹配。  
这可以防止运行时错误，并使您的代码更加健壮。  
`场景`：当将方法作为参数传递或存储以供后续调用时，您需要确保方法签名匹配。  
如果没有委托，您将缺乏类型安全，导致潜在的运行时错误和不太健壮的代码。

`解决方案(Solution)`：委托提供类型安全的函数引用，确保函数签名与委托签名相匹配。  
这可以防止运行时错误，并使代码更加健壮。

### 多点调用

多点调用是委托的一项功能，允许你使用单个委托调用多个方法。
这项功能在事件处理场景中很有用，当你想通知多个事件订阅者有关事件发生时。

`场景`：在事件处理中，您通常需要通知多个订阅者有关某个事件的发生。  
如果没有委托，您将不得不手动管理订阅者列表并调用它们的方法，这会导致复杂且容易出错的代码。

`解决方案(Solution)`：委托支持多点调用，允许单个委托引用多个方法。  
这简化了事件处理，通过自动管理订阅者列表并按顺序调用它们的方法。

## C# 中声明委托的最佳实践

在向类中添加委托时，遵循最佳实践非常重要，以确保代码组织良好、易于阅读且可维护。  
这里有一些关于在哪里定义委托的指南：

### 在类文件的顶部定义委托

开发人员通常在文件顶部声明委托，通常在命名空间内，但任何类的外部。

这种做法可以轻松找到委托的定义并理解其用途。

它还允许您定义可在同一命名空间内的多个类使用的委托。

示例： 

```c#
namespace MyNamespace

// Define the delegate outside the class
public delegate void MyDelegate(string message);

public class Publisher
{
    // Method that uses the delegate
    public void PublishMessage(MyDelegate del)
    {
        del("Hello from Publisher!");
    }
}

public class Subscriber
{
    public void Subscribe()
    {
        // Create an instance of the Publisher class
        Publisher publisher = new Publisher();

        // Create an instance of the delegate and pass a method to it
        MyDelegate del = new MyDelegate(PrintMessage);

        // Call the method of the Publisher class and pass the delegate
        publisher.PublishMessage(del);
    }

    // Method that matches the delegate signature
    public void PrintMessage(string message)
    {
        Console.WriteLine(message);
    }
}
```

### 在类中定义委托

如果一个委托仅限于某个类使用且不打算在类外部使用，你可以在类内部定义该委托。  
这种方式将委托封装在类内部，当委托与类的功能紧密相关且不打算在类外部使用时，这很有用。  
如果委托是公开的，同一命名空间内的其他类可以通过类的实例访问该委托。

示例： 

```c#
namespace MyNamespace

public class Publisher
{
    // Define a public delegate
    public delegate void MyDelegate(string message);

    // Method that uses the delegate
    public void PublishMessage(MyDelegate del)
    {
        del("Hello from Publisher!");
    }
}

public class Subscriber
{
    public void Subscribe()
    {
        // Create an instance of the Publisher class
        Publisher publisher = new Publisher();

        // Create an instance of the delegate and pass a method to it
        Publisher.MyDelegate del = new Publisher.MyDelegate(PrintMessage);

        // Call the method of the Publisher class and pass the delegate
        publisher.PublishMessage(del);
    }

    // Method that matches the delegate signature
    public void PrintMessage(string message)
    {
        Console.WriteLine(message);
    }
}
```

## 总结

委托是派生于`Delegate`类的.NET 类型，封装了方法。
- 委托允许你将方法作为参数传递，存储在变量中，并在运行时调用它们。
- 委托是类型安全的，这意味着编译器会在编译时检查方法签名是否与委托签名匹配。
- 委托可用于动态方法调用、回调方法、类型安全性和多点调用。
- 委托是 C#中事件处理的基础。
- 委托可以链接在一起，允许通过单个委托调用多个方法。
- 委托通常定义在类外部，以便其他类无需实例化类即可访问。
