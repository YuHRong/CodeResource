# 探索默认实现接口

`使用默认接口实现`，可以直接将新方法添加到接口中并提供默认实现。 使用此接口的类可以使用默认方法，也可以重写并提供自己的实现。

## 默认实现

接口中的默认实现允许向接口添加新方法，而不会造成问题。
- 接口可以使用默认实现为其方法提供主体。
- 使用接口的类不需要编写其自己的这些方法版本。
- 如果想要更改接口而不中断任何内容，此功能会有所帮助。

*注*:即使使用默认实现，类也不会自动获取接口的方法。 若要从接口使用方法，变量必须是接口类型。

### 优点

接口中的默认实现具有以下几个优点：
- `代码重用`：可以在不同实现之间共享通用函数，而无需复制代码。
- `向后兼容性`：无需使用接口更改所有类即可向接口添加新方法，以防止重大更改。
- `灵活性`：使用接口的类可以更改默认实现（如果需要不同的行为）。

例如，假设你正在使用客户管理系统，并添加了一个新方法，用于在现有界面中应用会员折扣。 默认方法计算基本折扣。 但是，如果“VIP 客户”类需要不同的计算，则可以修改此方法。

### 定义默认实现

在接口中定义默认实现的语法扩展接口声明以包含方法主体。 以下示例基于客户管理系统方案在接口中定义默认实现：

```c#
interface ICustomer
{
    int CustomerId { get; set; } // No default implementation

    void ApplyLoyaltyDiscount() 
    { 
        // default implementation
        Console.WriteLine("Applying 10% loyalty discount."); 
    }
}
```

在此示例中，`ApplyLoyaltyDiscount`方法在`ICustomer`接口中具有默认实现。 `CustomerId`属性也在接口中定义，但没有默认实现。

*注*从 C# 8.0 开始，接口可以包括默认方法实现。 更改允许将新方法添加到接口，而不会中断现有代码。 类可以使用这些默认值或提供自己的实现。

接下来，实施一个“VIPCustomer”类，该类需要进行不同的折扣计算。 此类可以实现`ICustomer`接口，为`CustomerId`属性提供自己的实现，并如下所示重写`ApplyLoyaltyDiscount`方法：

```c#
class VIPCustomer : ICustomer
{
    public int CustomerId { get; set; } // Implementing property from interface

    void ICustomer.ApplyLoyaltyDiscount()
    {
        // override default implementation for VIP customers
        Console.WriteLine($"Applying 20% loyalty discount for VIP customer with ID {CustomerId}.");
    }
}
```

在这种情况下，该`VIPCustomer`类应用 20% 折扣，而不是默认的 10%。 应用折扣时，`CustomerId`在消息中被使用。

在接口中，默认实现允许通过接口给出的方法实现来添加新方法。  
默认实现可促进代码重用、向后兼容性和灵活性。 它是 C# `Object-Oriented` 编程中的实用工具，可增强适应性，而不会中断现有代码。
