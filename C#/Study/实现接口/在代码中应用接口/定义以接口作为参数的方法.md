在 C# 中，接口为类建立协定。

## 接受接口的方法

C# 中的方法可以接受接口作为参数，以确保传递的对象遵守特定的协定。 将接口用作参数可以增强代码的灵活性和可重用性。  
例如，考虑接受`IAnimal`接口作为参数的方法：

```c#
public interface IAnimal
{
    void MakeSound();
}

public void MakeAnimalSound(IAnimal animal)
{
    animal.MakeSound();
}
```

在此示例中，`MakeAnimalSound`方法接受`IAnimal`接口，确保传递给它的任何对象都有一个`MakeSound`方法。
*注*将接口用作方法参数可促进松散耦合并提高可测试性。

## 示例：接口作为方法参数

在下面的示例中，我们有多个文件，每个文件包含不同的类或接口。 通常将项目放置在单独的文件中，以提高组织和可读性。

### IAnimal.cs

此文件定义IAnimal`接口，其中包括一个`MakeSound`方法。

IAnimal.cs
```c#
public interface IAnimal
{
    void MakeSound();
}
```

`Dog.cs`和`Cat.cs`

这些文件包含`Dog`和`Cat`类，它们可实现`IAnimal`接口并提供`MakeSound`方法。

`Dog.cs` 和 `Cat.cs`
```c#
public class Dog : IAnimal
{
    public void MakeSound()
    {
        Console.WriteLine("Woof!");
    }
}

public class Cat : IAnimal
{
    public void MakeSound()
    {
        Console.WriteLine("Meow!");
    }
}
```

### AnimalSound.cs

此文件包含`AnimalSound`类，它具有接受`IAnimal`接口作为参数并调用`MakeAnimalSound`方法的`MakeSound`方法。

AnimalSound.cs  
```c#
public class AnimalSound
{
    public void MakeAnimalSound(IAnimal animal)
    {
        animal.MakeSound();
    }
}
```

### Program.cs

此文件包含带有`Main`方法的`Program`类，该方法创建`Dog`和`Cat`的实例，并使用`AnimalSound`类使它们发出声音。

Program.cs
```c#
public class Program
{
    public static void Main()
    {
        IAnimal dog = new Dog();
        IAnimal cat = new Cat();
        
        AnimalSound animalSound = new AnimalSound();
        animalSound.MakeAnimalSound(dog); // Output: Woof!
        animalSound.MakeAnimalSound(cat); // Output: Meow!
    }
}
```

上面示例演示如何将接口用作方法参数，促进松散耦合并提高可测试性。 这是了解 C# 中接口概念的简单但有效的方法。
