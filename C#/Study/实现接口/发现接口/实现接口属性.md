# 实现接口属性

作为负责创建项目的接口的开发人员，可以通过设计具有`属性`的接口来启动该过程。 然后，开发实现这些属性的类。

C# 中的接口类似于蓝图。 它概述了类想要使用此接口时需要具有的功能（属性或方法）。 对于此项目，定义仅定义属性的接口（没有方法）。

- 使用`interface`关键字创建接口。
- 在接口中定义没有正文的属性。
- 接口可以选择为成员（包括属性）提供默认实现，但这种情况很少发生。

```c#
public interface IVehicle
{
    int Speed { get; set; }
    string Color { get; set; }
}
```

上面代码为`IVehicle`创建了具有`Speed`属性和`Color`属性的接口。

接口属性通常没有任何代码，它们只需指定数据类型（例如`int`或`string`）以及是否可以从它们`读取`、`写入`或同时执行两者。   
在类中实现这些属性时，你将编写用于获取（读取）和设置（写入）值的实际代码。 例如，如果接口具有`Speed`属性，你将决定如何在类中存储和检索速度值。

## 开发实现接口的类

当一个类（如`Car`）决定实现接口时，`IVehicle`就像说“我要生成一辆车”。必须`Car`包括蓝图中`IVehicle`概述的所有功能（属性或方法）。

- 语法`Car : IVehicle`用于指示正在`Car`实现`IVehicle`接口。 使用熟悉的语法，与用于继承的语法相同。
- 必须为`Car`蓝图中列出的`IVehicle`所有功能提供公共的非静态实现。
- 如果`Car`决定实现具有类似功能的多个蓝图（接口），则需要清楚地了解哪个功能来自哪个蓝图。

下面介绍如何从`Car`在代码中实现`IVehicle`接口：

```c#
public class Car : IVehicle
{
    private int _speed;
    private string _color;

    public int Speed
    {
        get { return _speed; }
        set { _speed = value; }
    }

    public string Color
    {
        get { return _color; }
        set { _color = value; }
    }
}
```

`注意`符号`:`用于指示`Car`实现`IVehicle`，类似于用于指示继承的方式`:`。

在此示例中，`Car`是一个实现`IVehicle`接口并提供其自己的版本和`Speed`版本的`Color`类。 就像基于蓝图构建车辆一样，`IVehicle`而是为速度和颜色添加自己的独特触感。
