# 创建接口方法

作为负责创建项目的接口的开发人员，除了仅具有`属性`的接口之外，还开始添加接口`方法`。 若要测试接口，请开发实现这些方法的类。

接口是类必须实现的一组相关函数。 它可以包括方法、属性、事件和索引器。  
例如，考虑一个命名为`IVehicle`的接口，它包含`Drive`方法以及`Speed`和`Color`属性：

```c#
public interface IVehicle
{
    int Speed { get; set; }
    string Color { get; set; }
    void Drive();
}
```

`注意`如何在接口中没有正文的情况下声明`Drive`方法。 将方法添加到接口时，请在接口声明中包括方法的签名，但不包括其正文。 实现接口的类提供方法的主体。

## 构造实现接口的类

对于具有属性、方法或两者的类，在类中实现接口是相同的。 创建实现接口的类时，请遵循以下准则：
- 为所有接口的成员提供实现。
- 将接口声明为类定义的一部分，并提供所需的方法主体。

`注意`类可以实现多个接口，从而允许它包含来自多个源的行为。 但是，类只能继承自单个类。 如果基类实现接口，则从基类派生的任何类都继承该实现。

请查看以下`Car`类示例，该示例实现`IVehicle`接口。

```c#
public class Car : IVehicle
{
    private int _speed;
    private string _color;

    public int Speed
    {
        get { return _speed; }
        set { _speed = value; }
    }

    public string Color
    {
        get { return _color; }
        set { _color = value; }
    }

    public void Drive()
    {
        // Implementation of the Drive method
        Console.WriteLine("The car is driving.");
    }
}
```

在此示例中，`Drive`该方法实现为在调用时显示消息。 这只是一个简单的实现，用于演示目的。 在实际应用程序中，该方法`Drive`可能包含更复杂的代码，用于控制汽车驾驶方式。

接口定义实现类所需的方法签名和属性协定。 当类实现接口时，它为这些方法提供实际代码，从而有效地将方法签名转换为功能例程。 此过程允许类通过实现多个接口来采用多种行为。 在接口中继续开发和定义方法时，你会发现，使用接口实际上可能会导致更简洁、更高效且易于管理的代码。
