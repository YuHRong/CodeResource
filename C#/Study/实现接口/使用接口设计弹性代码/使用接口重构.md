先看示例： 

```c#
public class BorrowableBook
{
    public string Title { get; set; }
    public bool IsAvailable { get; private set; } = true;

    public BorrowableBook(string title)
    {
        Title = title;
    }

    public void Borrow()
    {
        if (IsAvailable)
        {
            IsAvailable = false;
            Console.WriteLine($"You have borrowed \"{Title}\".");
        }
        else
        {
            Console.WriteLine($"\"{Title}\" is already borrowed.");
        }
    }
}

public class Library
{
    private BorrowableBook _book;

    public Library(BorrowableBook book)
    {
        _book = book;
    }

    public void BorrowBook()
    {
        if (_book.IsAvailable)
        {
            _book.Borrow();
        }
        else
        {
            Console.WriteLine("The book is not available.");
        }
    }
}
```

这种设计将 `Library` 类与 `BorrowableBook` 类紧密耦合，导致在不修改 `Library` 类的情况下难以添加新的可借用类型（`例如 DVD`）。

为了解决紧密耦合问题，我们引入一个接口来解耦借用行为。

## 引入接口

接口为行为设定契约，但不详细说明其实现。
引入接口抽象了借用功能，并提高了系统的灵活性。

这是`IBorrowable`接口：

```c#
public interface IBorrowable
{
    bool IsAvailable { get; }
    void Borrow();
}
```

这个接口引入了两个成员：
- `IsAvailable`：一个属性，表示该物品是否可供借用。
- `Borrow`: 一种借用文件的方法。

通过定义这个接口，我们将借用行为与任何特定实现解耦。

## 重构 BorrowableBook 类

接下来，我们将`BorrowableBook`类更新为实现`IBorrowable`接口：

```c#
public class BorrowableBook : IBorrowable
{
    public string Title { get; set; }
    public bool IsAvailable { get; private set; } = true;

    public BorrowableBook(string title)
    {
        Title = title;
    }

    public void Borrow()
    {
        if (IsAvailable)
        {
            IsAvailable = false;
            Console.WriteLine($"You have borrowed \"{Title}\".");
        }
        else
        {
            Console.WriteLine($"\"{Title}\" is already borrowed.");
        }
    }
}
```

现在， `BorrowableBook` 类遵循 `IBorrowable` 接口，因此它可以与其他实现相同接口的类互换。

## 重构 Library 类

我们还更新`Library`类，使其依赖于`IBorrowable`接口而不是具体的`BorrowableBook`类：

```c#
public class Library
{
    private IBorrowable _item;

    public Library(IBorrowable item)
    {
        _item = item;
    }

    public void BorrowItem()
    {
        if (_item.IsAvailable)
        {
            _item.Borrow();
        }
        else
        {
            Console.WriteLine("The item is not available.");
        }
    }
}
```

现在， `Library` 类可以与任何实现 `IBorrowable` 接口的对象协同工作，使其更加灵活且易于扩展。

## 使用依赖注入

想象你正在设置家庭娱乐系统。
你不会永久将特定品牌的音箱连接到你的立体声设备上，而是使用可以接受多种兼容音箱插头的"接口"。
这种设计让你可以轻松地更换或升级音箱，而无需更换整个立体声设备。

在软件中，依赖注入的工作方式类似。
它允许一个类依赖于抽象接口而不是特定的实现。
依赖注入使系统更加灵活且易于维护，因为你可以“插入”不同的实现而无需修改类本身。

在编程中，构造函数就像在设置你的立体音响系统时连接扬声器插孔的技术人员。

对于`Library`类，构造函数（`public Library(IBorrowable item)`）是提供依赖的地方。构造函数允许`Library`类与任何兼容的实现（如`BorrowableBook`或`BorrowableDVD`）一起工作，而无需更改其内部结构。

就像扬声器插孔能够在选择不同扬声器方面提供灵活性一样，构造函数在使用各种可借用的项目方面也提供了灵活性。

在代码中的实现方式： 

```c#
public class Library
{
    private IBorrowable _item;

    public Library(IBorrowable item) // Dependency is injected here
    {
        _item = item;
    }

    public void BorrowItem()
    {
        if (_item.IsAvailable)
        {
            _item.Borrow();
        }
        else
        {
            Console.WriteLine("The item is not available.");
        }
    }
}
```

在这个示例中： 
- 构造函数是建立"连接"（`IBorrowable`依赖）的地方，类似于演讲插头连接到"插孔"的方式。
- `IBorrowable`接口就像“音频插孔”，定义了标准的连接点。
- `BorrowableBook`和`BorrowableDVD`就像不同类型的立体声音箱，具有独特的连接器，这些连接器连接到“插孔”。
- 通过使用依赖注入，`Library`类可以与`IBorrowable`的任何实现一起工作。

这种方法提供了：
- `灵活性`：您可以轻松切换或添加新的实现，而无需修改`Library`类。
- `可测试性`：你可以为测试目的插入模拟实现。
- `可维护性`：`Library`类不需要知道具体实现的细节，这使得它更容易扩展和维护。

这种设计确保`Library`类不再与特定实现紧密耦合，使系统更具模块化和适应性。

## 添加新的可借用项

有了接口，我们可以轻松添加新的可借用类型，而无需修改`Library`类。例如，这是一个`BorrowableDVD`类：

```c#
public class BorrowableDVD : IBorrowable
{
    public string Title { get; set; }
    public bool IsAvailable { get; private set; } = true;

    public BorrowableDVD(string title)
    {
        Title = title;
    }

    public void Borrow()
    {
        if (IsAvailable)
        {
            IsAvailable = false;
            Console.WriteLine($"You have borrowed the DVD \"{Title}\".");
        }
        else
        {
            Console.WriteLine($"The DVD \"{Title}\" is already borrowed.");
        }
    }
}
```

`BorrowableDVD`类实现了相同的`IBorrowable`接口，因此它可以与`Library`类无缝使用。

## 测试系统

这是一个演示重构系统的程序：

```c#
using System;

class Program
{
    static void Main()
    {
        // Create borrowable items
        IBorrowable book = new BorrowableBook("Adventure Works Cycles");
        IBorrowable dvd = new BorrowableDVD("Graphic Design Institute");

        // Create libraries
        Library bookLibrary = new Library(book);
        Library dvdLibrary = new Library(dvd);

        // Borrow items
        bookLibrary.BorrowItem();
        bookLibrary.BorrowItem(); // Try borrowing again

        Console.WriteLine();

        dvdLibrary.BorrowItem();
        dvdLibrary.BorrowItem(); // Try borrowing again
    }
}
```

输出展示了重构系统的灵活性：

```
You have borrowed "Adventure Works Cycles".
"Adventure Works Cycles" is already borrowed.

You have borrowed the DVD "Graphic Design Institute".
The DVD "Graphic Design Institute" is already borrowed.
```

这个重构的示例展示了接口如何减少依赖并提高模块化：
- `关注点分离`：`IBorrowable`接口隔离了借用行为，确保`Library`类不依赖于特定的实现。
- `增强灵活性`：你可以添加新的可借用类型（例如，DVD），而无需修改`Library`类。
- `简化维护`：由于职责划分明确，系统更易于理解、测试和扩展。
