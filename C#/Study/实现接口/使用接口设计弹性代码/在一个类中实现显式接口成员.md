在 C#中，显式接口实现允许一个类以一种方式实现接口成员，即它们只能通过接口访问，而不能通过类实例访问。

## 一个类中的多个接口

在 C#中，一个类可以实现多个接口，从而可以从多个来源继承行为。  
结合多个接口的能力至关重要，因为 C#不支持类的多重继承。  
实现多个接口允许一个类拥有不同的特性，并确保它遵循这些接口设定的规则。  

接口定义了一组相关的功能。
- 类或结构必须实现它所继承的所有接口成员。
- 接口可以包含方法、属性、事件和索引器。

下面示例有两个简单接口：

``` c#
public interface IEnglishDimensions
{
    float Length();
    float Width();
}

public interface IMetricDimensions
{
    float Length();
    float Width();
}
```

## 显式接口成员

在 C#中，一个类只能继承自一个类，但可以实现多个接口。

当多个接口具有相同的成员名时，这一点尤其重要。  
为了避免混淆，我们使用显式接口实现。

显式接口实现确保接口的每个成员都有唯一的实现，防止冲突。

它还明确了当一个类实现两个或多个具有相同成员名的接口时，成员属于哪个接口。

显式接口实现允许一个类满足多个接口的要求，提供多样化的功能。

要显式实现接口的成员，你需要在成员名之前添加接口名和一个点。  
示例

```c#
public class Box : IEnglishDimensions, IMetricDimensions
{
    float IEnglishDimensions.Length()
    {
        return 30.0f;
    }

    float IEnglishDimensions.Width()
    {
        return 20.0f;
    }

    float IMetricDimensions.Length()
    {
        return 76.2f;
    }

    float IMetricDimensions.Width()
    {
        return 50.8f;
    }
}
```

在这段代码中，`Box`类正在实现两个接口：`IEnglishDimensions`和`IMetricDimensions`。这两个接口都有名为`Length`和`Width`的成员。为了避免混淆，每个成员都明确地与其对应的接口关联。

*注*显式实现的接口成员没有`public`、`private`等访问修饰符。它们只能通过接口的实例访问，而不是通过类。  
这意味着你不能直接调用`Box.Length()`，但如果你有一个被当作`IEnglishDimensions`处理的`Box`的实例，你可以调用`IEnglishDimensions.Length()`。

## 使用显式接口成员

你需要创建接口的实例并调用它们的成员来执行和测试使用显式接口成员的代码。  
这确保了根据接口类型调用正确的实现。

```c#
Box box = new Box();
IEnglishDimensions eDimensions = box;
IMetricDimensions mDimensions = box;

Console.WriteLine($"Length in inches: {eDimensions.Length()}");
Console.WriteLine($"Width in inches: {eDimensions.Width()}");
Console.WriteLine($"Length in centimeters: {mDimensions.Length()}");
Console.WriteLine($"Width in centimeters: {mDimensions.Width()}");
```

在这个示例中，`Box`类实例被转换为`IEnglishDimensions`和`IMetricDimensions`以访问显式接口成员。

输出显示的尺寸既有英文单位也有公制单位。

```
Length in inches: 30
Width in inches: 20
Length in centimeters: 76.2
Width in centimeters: 50.8
```

在 C#中，你可以在类中实现多个接口，使用显式接口实现来在成员具有相同签名时解决命名冲突。  
显式实现的成员通过接口实例访问，而不是类实例，确保了正确的功能并避免了歧义。
