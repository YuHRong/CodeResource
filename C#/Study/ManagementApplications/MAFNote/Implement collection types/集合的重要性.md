在 C#中，集合对于高效管理对象组至关重要。  
它们提供了一种灵活的方式来存储、检索和操作数据，这对开发者在开发各种应用程序时至关重要。

## 集合在管理对象组中的重要性

C#中的集合允许开发者通过将数据作为一组进行存储和操作，从而更高效地处理相似数据。 这种能力对于需要动态数据处理的程序至关重要，例如添加或删除元素。

集合可以根据它们访问元素的方式、性能特征以及动态增长或缩小的能力进行分类。<br>例如，某些集合通过索引访问元素，如`List<T>`，而另一些则使用键，如`Dictionary<TKey, TValue>`。
- 集合是强类型的，这意味着它们存储特定类型的元素（`T`），确保类型安全并减少运行时错误。
- 集合提供添加、删除或查找项目的方法。
- 它们支持枚举，允许使用循环遍历元素。
- 集合可以复制到数组中，提供数据操作的灵活性。

*注*:例如，您可能需要为`System.Collections.Generic`和`System.Linq`命名空间添加 `using` 指令。

## 描述 `List<T>`、`HashSet<T>`和` Dictionary<TKey, TValue>`

`List<T>`是一个可索引的集合，允许通过位置访问元素。<br>它支持动态调整大小，适用于元素数量可能变化的场景。

`HashSet<T>`是一个为高性能集合操作设计的集合。<br>它不允许重复元素，并且适用于需要唯一性的场景。

`Dictionary<TKey, TValue>`将元素存储为键值对，可通过键快速查找。<br>当你需要将值与唯一键关联起来时，这很有用，例如在银行应用程序中将客户 ID 映射到账户详情。

## 为银行场景中集合的应用做准备

在银行应用程序中，集合在高效管理数据方面发挥着关键作用。<br>例如，一个`Dictionary<int, Account>`可以将客户 ID 映射到其相应的账户详情，从而实现快速检索和更新。<br>类似地，一个`List<Transaction>`可以存储一系列交易记录，支持使用 LINQ（语言集成查询）进行排序和过滤等操作。
- 使用`Dictionary<TKey, TValue>`快速查找客户数据。
- 使用`List<T>`来记录有序的交易记录。
- 利用`HashSet<T>`来确保条目的唯一性，例如客户 ID。
